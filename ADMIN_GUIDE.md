# Руководство администратора

## Роли

### Admin

Может:

- управлять пользователями и ролями;
- видеть все заявки, платежи, документы и аудит;
- менять настройки филиалов и услуг;
- экспортировать данные;
- управлять интеграциями.

### Manager

Может:

- обрабатывать заявки;
- закреплять менеджера;
- планировать консультации;
- проверять документы;
- вести запись на практику;
- смотреть платежи и статусы учеников.

### Student

Может:

- видеть свой профиль;
- смотреть прогресс обучения;
- загружать документы;
- записываться на практику;
- видеть свои платежи и уведомления.

## CRM-процесс

1. Заявка приходит с сайта, мобильного приложения или Telegram.
2. Менеджер проверяет контакт и категорию.
3. Менеджер назначает консультацию.
4. Ученик передаёт документы.
5. Создаётся договор и график оплаты.
6. После оплаты ученик получает доступ к LMS и практике.
7. Менеджер отслеживает прогресс, документы, платежи и экзамены.

## Текущий статус админки

Админка уже содержит рабочий интерфейс:

- dashboard metrics;
- поиск клиентов;
- фильтр по статусам;
- таблица лидов;
- блок практики;
- блок платежей;
- responsive layout.

Но сейчас данные демонстрационные. До production нужно:

- подключить Firebase Auth;
- включить RBAC через custom claims;
- заменить sample data на Firestore queries;
- добавить реальные действия кнопок;
- добавить audit logs;
- добавить обработку ошибок и empty/loading states для реальных запросов.

## Как запустить админку

```bash
npm run dev:admin
```

URL:

```text
http://localhost:3001
```

## Как добавить филиал

Филиалы сейчас лежат в:

```text
packages/shared/src/index.ts
```

Массив:

```ts
branches;
```

После подключения Firestore филиалы лучше перенести в коллекцию `branches`, а в коде оставить только fallback/default data.

## Как изменить услуги и цены

Услуги сейчас лежат в:

```text
packages/shared/src/index.ts
```

Массив:

```ts
services;
```

Цены указаны как `priceFrom` и отражают стоимость теоретической части. Практические занятия оплачиваются отдельно по тарифу города.

## Как работать с документами

Плановый путь Storage:

```text
student-documents/{studentId}/{fileName}
```

Допускаются:

- PDF;
- изображения;
- размер до 10 МБ.

## Что важно не забыть

- Не выдавать роль `admin` обычным менеджерам.
- Не хранить секреты в `.env.example`.
- Не открывать Firestore collections без явной причины.
- Не включать demo payment adapter в production.
- Не принимать Telegram webhook без `TELEGRAM_WEBHOOK_SECRET`.
