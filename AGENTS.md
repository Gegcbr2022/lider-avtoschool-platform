# AGENTS.md — оркестрация AI-агентов проекта «Лідер»

Монорепо npm-workspaces. Мобильное приложение (`apps/mobile`, Expo Router) — главный продукт.
Источник истины для mobile — **установленный APK через ADB**, не предположения.

## Роли
- **Opus 4.8** — архитектура, аудит, приёмка эпиков перед merge. Промт: `docs/OPUS_REVIEW_PROMPT.md`.
- **Codex** — логика/безопасность/push/платежи/бронь/рейтинг-логика. Промт: `docs/AGENT_TASKS_CODEX.md`.
- **Claude Sonnet** — UI/UX/темы/тексты/анимации/empty-states. Промт: `docs/AGENT_TASKS_SONNET.md`.

## Порядок работы
1. Codex — своя ветка/эпик, маленькие коммиты, не трогает уже исправленное Opus.
2. Sonnet — после Codex, только UI/контент по тем же файлам, без бизнес-логики.
3. Opus — приёмка (verify командами + ADB), затем merge.

## Незыблемые правила
- Не ломать функционал; не хардкодить ключи; секреты только в env (`.env.example`).
- Не трогать prod Firebase опасно без явного чекпоинта; не удалять большие файлы без списка+причины.
- Украинский основной; **никаких мок-данных на проде**; Monobank-логика ≤3 нажатия.
- После блока: `typecheck`/`lint`/`test` (+ `test:rules`); mobile → пересборка APK + ADB-smoke + logcat без FATAL.
- Каждому агенту: **искать смежные баги**, а не только заданные; в конце — отчёт + следующий промт.
