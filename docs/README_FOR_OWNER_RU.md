Технические детали — в соседних файлах `docs/`.

## 1. Что сейчас с проектом

Приложение работает и стабильно (на эмуляторе не падает). Безопасность закрыта (вход под защитой,
2FA для админов, бэкапы). Что я поправил в этой сессии:
- **Убрал вечную «1» на иконке Профиля** (это были авто-напоминания «тест дня/серія», а не реальные сообщения).
- **Переделал маленькие кнопки на главной** («Тренажер/Лідік/Чат/Клуб») в крупные красивые карточки 2×2.

Эти правки сделаны в коде. Чтобы увидеть их на телефоне — нужно **пересобрать APK** (см. п.4).

## 2. Что просит сделать ВАС (без этого код стоит)

Коротко (полностью — в `docs/FIREBASE_SETUP_CHECKLIST.md`):
1. **App Check**: в Firebase поставить режим **Monitor** (пока не блокировать), добавить Play Integrity
   и debug-token для тестового устройства. Сейчас на эмуляторе ошибка 403 — это ожидаемо без токена.
2. **Ключ подписи для Google Play** (сейчас приложение подписано тестовым ключом — для магазина нужен боевой).
3. **Эквайринг** (приём оплат): подключить Monobank Acquiring — нужны ваши ключи/мерчант.
4. **Контент**: ссылки на видео-уроки и полный текст ПДР по разделам.
5. **Партнёры**: страховщик, юрист, СТО/мойки (для разделов страховки/юриста/сервисов).
6. **Email-шаблоны** (подтверждение/сброс пароля) — оформить под бренд, чтобы не уходили в спам.
7. **Telegram-бот**: права на рассылку + включить Topics (мост уже готов в коде).

> Я НЕ прошу вас делать то, что можно сделать кодом. Этот список — только внешние сервисы/ключи/контент.

## 3. Как запускать (разработка)

```powershell
# из корня C:\Avtoschool_APP
npm install                     # один раз
npm run dev:web                 # сайт
npm run dev:admin               # админка (CRM)
npm run dev:api                 # API (Firebase Functions локально)
npm run dev:mobile              # мобильное (Expo)
```

## 4. Как пересобрать мобильный APK и проверить

```powershell
$env:EXPO_NO_METRO_WORKSPACE_ROOT=1; $env:NODE_ENV="production"
cd apps\mobile\android
.\gradlew assembleRelease
# установить на эмулятор/устройство:
adb install -r -d -t app\build\outputs\apk\release\app-release.apk
```
Затем открыть приложение: на **Главной** — крупные карточки 2×2; на **Профиле** — иконка без «1»
(если нет реальных сообщений менеджера/брони). Подробный QA — `docs/ANDROID_QA_ADB.md`.

## 5. Проверки качества (перед релизом)

```powershell
npm run typecheck      # типы
npm run lint           # стиль
npm run test           # тесты (+ smoke)
npm run test:rules     # правила Firestore (нужен эмулятор)
npm run build          # сборка всех приложений
```
